# backendNewReport Fix Plan

## Goal

Address the actionable findings from `docs/backendNewReport.json` without chasing analyzer noise. The report shows `77/C`, `14` security issues, `6` high security issues, duplicate helpers, and layer violations. Manual cross-check indicates most high SQL injection findings are false positives because the flagged queries use parameter placeholders. The practical fixes are logging hygiene and small DRY refactors.

## Recommended Scope

1. Sanitize OTP logging in `backend/src/modules/auth/auth.controller.js`.
  - Remove direct `console.log` and `console.error` calls from `sendOtp` and `verifyOtp`.
  - Do not log `req.body`, `req.validated`, raw phone numbers, raw OTPs, Redis OTP payloads, or stack traces.
  - Use existing `logger` with masked phone numbers and event names only.
  - Keep `devOTP` response behavior unchanged unless product wants it removed.
2. Extract shared `addressToText`.
  - Add `backend/src/utils/address.js` with the existing behavior:
   string input returns as-is, `text`/`addressText` wins, otherwise join `line1`, `line2`, `city`, `state`, `pincode`.
  - Replace local copies in:
    - `backend/src/modules/admin/admin.controller.js`
    - `backend/src/modules/delivery/delivery.controller.js`
    - `backend/src/modules/orders/orders.controller.js`
3. Extract shared status transition helper.
  - Add `backend/src/utils/orderStatus.js`.
  - Export two transition maps because delivery and order controllers intentionally allow different transitions:
    - `ORDER_STATUS_TRANSITIONS`
    - `DELIVERY_STATUS_TRANSITIONS`
  - Export `canTransition(transitions, fromStatus, toStatus)`.
  - Replace local `canTransition` functions in:
    - `backend/src/modules/delivery/delivery.controller.js`
    - `backend/src/modules/orders/orders.controller.js`
4. Optional analyzer-noise cleanup.
  - Only if the report score must improve, refactor dynamic allowlisted update builders in `backend/src/modules/admin/admin.controller.js` for category/product patch paths.
  - Keep behavior identical and values parameterized.
  - Do not prioritize this as security-critical unless user-controlled column identifiers are introduced.

## Non-Goals

- Do not refactor large files broadly in this pass.
- Do not change SQL statements that are already parameterized only to satisfy a heuristic.
- Do not address reported layer violations unless a real import direction issue is manually confirmed.
- Do not remove CLI `console` output from smoke/e2e/migration scripts.
